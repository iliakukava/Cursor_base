# Инструкция — Модуль 02. Контекст

> Делай каждый шаг в Cursor параллельно с чтением. Не просто читай — пробуй.

---

## Шаг 1. Почему AI «тупит»

Открой AI Chat (`Cmd+L` на Mac, `Ctrl+L` на Windows) и напиши:

```
Напиши текст для главной страницы
```

Результат будет абстрактный и бесполезный. AI не знает кто ты, что за бизнес, для кого страница. Это не AI тупой — это ты не дал ему контекст.

**Запомни этот результат** — в конце модуля мы повторим этот запрос, и разница тебя удивит.

---

## Шаг 2. Что такое контекст

Контекст — это всё, что AI знает о тебе, твоём проекте и задаче прямо сейчас.

Представь: ты нанял стажёра с IQ 150. Он гений, но в первый рабочий день не знает ничего: ни что за компания, ни кто клиенты, ни как тут принято общаться. И каждое утро он просыпается с чистой памятью.

Вот это и есть AI без контекста. Твоя задача — дать ему «память».

**Правило:** 90% качества ответа зависит от контекста, который ты дал. Не от промпта. От контекста.

---

## Шаг 3. Система папок = мозг AI

У AI нет долговременной памяти. Его память — это файлы и папки в твоём проекте.

Сравни:

**❌ Хаос:**
```
Рабочий стол/
├── Без имени.docx
├── отчет_финал_ФИНАЛ2.xlsx
├── IMG_4820.png
└── аыфвоа.txt
```

**✅ Система:**
```
Проект/
├── 00_Inbox/
├── 01_Продукт/
├── 02_Маркетинг/
├── 03_Продажи/
└── 99_Системное/
```

Правильная структура папок = AI понимает контекст без промптов. Он видит имена папок и файлов — и уже понимает, о чём речь.

**Запомни:** Если чего-то нет в файле — этого не существует для AI.

---

## Шаг 4. Создай структуру для жизни

Начни с личного контекста — не с бизнеса. AI должен знать тебя как человека.

1. Создай новую папку и открой её в Cursor
2. Нажми `Cmd+I` (Composer)
3. Напиши:

```
Создай структуру папок для моей жизни.
Я хочу чтобы AI помогал мне во всём:
здоровье, учёба, путешествия, финансы, хобби.
Спроси что мне важно.
В каждой папке — README.md
```

AI задаст вопросы и создаст структуру под тебя. Примерный результат:

```
МОЯ_ЖИЗНЬ/
├── 00_Inbox/
├── 01_Здоровье/
│   ├── анализы/
│   ├── тренировки/
│   └── питание/
├── 02_Учёба/
│   └── курсы/
├── 03_Путешествия/
├── 04_Финансы/
├── 05_Хобби/
└── 99_Системное/
```

**00_Inbox** — это «приёмная». Кидай сюда всё, что не знаешь куда положить. Раз в неделю попроси AI: «Разложи файлы из Inbox по папкам».

---

## Шаг 5. «Кабинетная система»

Запомни метафору — она поможет понять логику:

- **Папка** = Кабинет (отдел компании)
- **Файл** = Документ на столе в кабинете
- **README.md** = Табличка на двери («Что здесь лежит»)
- **00_Inbox** = Приёмная (бросай всё сюда, AI разложит)

Когда AI открывает твой проект — он как новый сотрудник, который ходит по кабинетам и читает таблички на дверях. Чем лучше таблички (README) — тем быстрее он разберётся.

---

## Шаг 6. README.md — карта навигации

README — это первое, что читает AI, когда открывает папку. Один файл вместо сотни.

**Что писать в README:**
- 🎯 О папке — зачем она нужна
- 📂 Что тут хранить — какие файлы
- ⚙️ Как использовать — что делать с содержимым

**Пример для папки «03_Продажи»:**
```markdown
## 03_Продажи

Всё про продажи: скрипты, CRM, отчёты, воронки.

CRM: AmoCRM
Менеджеров: 3
Средний чек: 45 000 ₽
```

**Попробуй прямо сейчас:**

В Composer (`Cmd+I`) напиши:
```
Создай README.md в каждой папке проекта с описанием что там хранить
```

## Шаг 7. Контекст = Тело / Разум / Дух

Три слоя, чтобы AI понял «кто ты» — и давал ответы под тебя, а не общие шаблоны.

**🏃 Тело:**
- Анализы, чекапы
- Спорт (что любишь, какие травмы)
- Сон, энергия, вес
- Что нельзя (аллергии, ограничения)

**🧠 Разум:**
- Психотесты (MBTI, Big Five)
- Стиль принятия решений
- Что бесит (триггеры)
- Книги + почему именно они

**✨ Дух:**
- Цель на 3 месяца
- Стратегия жизни
- Ценности
- Куда идёшь

Создай файл `КОНТЕКСТ_О_СЕБЕ.md` и заполни по этим трём блокам. Не нужно писать роман — по 3-5 пунктов в каждом блоке достаточно.

## Шаг 8. Тесты — psytests.org

Пройди тесты и скинь результаты в папку. AI проанализирует и соберёт профиль.

**Обязательные:**
- **Big Five** — 5 черт личности (открытость, добросовестность, экстраверсия, доброжелательность, нейротизм)
- **MBTI (16 типов)** — как ты думаешь и принимаешь решения
- **PHQ-9** — уровень энергии
- **GAD-7** — уровень тревожности

**Бонусные:**
- Стиль решений — аналитик или интуит
- Эмоциональный интеллект

Сайт: **psytests.org** — бесплатный, на русском.

**Что делать:**
1. Пройди тест
2. Сделай скрин результатов (или скопируй текст)
3. Положи в папку `02_Тесты/`
4. Попроси AI: «Проанализируй мои тесты и собери профиль личности»

Результат: AI узнает твой психотип и начнёт общаться как личный коуч, а не как Википедия.

## Шаг 9. Вкусы — по 3 штуки

Этого хватит, чтобы AI понял, как ты думаешь и что тебе нравится.

Запиши в файл (или прямо в контекст):
- 3 любимых **фильма**
- 3 любимых **сериала**
- 3 **книги** + почему именно они (одна мысль из каждой)
- 3 **артиста / группы**
- 3 **страны / города**
- 3 **любимых бренда**
- 3 вещи, которые **бесят**
- 3 **подкаста / канала**

**Важно:** Не умничай. Пиши как есть. Книги — не «Война и мир», а то что реально зацепило.

Это меняет стиль ответов AI «под тебя». Попробуй — попроси AI написать тебе пост после загрузки вкусов. Разница будет заметна.

## Шаг 10. Как скормить AI

Простой протокол — 3 шага:

1. **Собери всё** в одну папку (анализы, скрины тестов, заметки о себе)
2. **Подключи папку** через `@` в чате Cursor
3. **Промпт:**

```
@02_Личное

Собери мой профиль: Тело / Разум / Дух.
Коротко и конкретно. Выведи как файл PROFILE_FOR_AI.md
```

AI соберёт всё в один файл, который потом можно подключать к любому запросу.

**⚠️ НЕ грузи:** паспорт, пароли, банковские данные. Только контекст для работы.

## Шаг 11. Бизнес-контекст

Теперь то же самое — но для бизнеса. AI должен знать о твоём бизнесе всё, один раз.

Создай папку с описанием бизнеса:

**Что описать:**
- Чем занимается компания — сфера, услуги, продукт
- Целевая аудитория — кто клиенты, их боли
- Команда — сколько человек, роли
- Тон общения — формальный, дружелюбный, экспертный
- Конкуренты — от кого отличаемся

**Пример структуры (на примере АССИСТ+):**
```
МОЙ_БИЗНЕС/
├── README.md
├── 01_О_компании/
│   ├── описание.md      ← "HR-агентство по подбору бизнес-ассистентов"
│   └── услуги.md        ← что конкретно делаем
├── 02_Клиенты/
│   └── портрет_ЦА.md    ← предприниматели с выручкой от 1 млн/мес
├── 03_Маркетинг/
│   └── контент_план.md
└── 00_Inbox/
```

**Промпт для создания:**
```
Я занимаюсь [опиши бизнес в 2-3 предложениях].
Создай структуру папок для бизнес-контекста.
В каждой папке — README.md с описанием.
```

## Шаг 12. Протестируй контекст

3 теста, которые покажут — понимает ли AI тебя:

**Тест 1: «Кто я?»**
```
Опиши мой бизнес в одном предложении. Кто мои клиенты?
```
Если AI отвечает общими фразами — контекст слабый.

**Тест 2: «Напиши пост»**
```
Напиши пост для Telegram в моём стиле
```
Если стиль чужой — AI не знает тебя.

**Тест 3: «Бизнес-совет»**
```
Клиент просит скидку 30%. Что мне ответить?
```
Если AI не знает твоих цен и политику — контекст пустой.

Не прошёл тесты? Вернись к шагам 7-11 и добавь больше контекста.

## Шаг 13. Символ @ — подключение контекста

Символ `@` в Cursor — это способ сказать AI: «Посмотри сюда!»

В AI Chat (`Cmd+L`) набери `@` — появится выпадающий список:

**@Codebase** — подключает весь проект. AI видит все файлы.
```
@Codebase Напиши текст для главной страницы
```
Когда использовать: нужен обзор всего проекта, рефакторинг, поиск связей.
Когда НЕ использовать: мелкие задачи в одном файле (будет медленнее).

**@Web** — свежая информация из интернета.
```
@Web Какие AI-инструменты для бизнеса появились в 2026 году?
```
Когда использовать: актуальные цены, новые инструменты, проверка фактов.

**@Doc** — документация внешних сервисов.
```
@Doc Telegram Bot API Как отправить фото через бота?
```
Как добавить свою документацию: Settings → Features → Docs → Add new doc → вставь URL.

**@Git** — история изменений проекта.
```
@Git Какие изменения были за последнюю неделю?
```
Полезно: разобраться что менялось, найти причину бага, написать changelog.

## Шаг 14. .cursorrules — правила, которые AI читает всегда

`.cursorrules` — файл в корне проекта, который Cursor читает автоматически при каждом запросе. Тебе не нужно подключать его через `@` — он работает всегда.

Ты уже создал его в Модуле 01. Теперь добавь туда контекст:

```
# Правила проекта

## О проекте
[Кратко: что это, для кого, зачем]

## Стек
[Какие технологии используются]

## Стиль
- Тон: [формальный / дружелюбный / экспертный]
- Язык: русский
- Не использовать: [что избегать]

## О пользователе
[2-3 предложения о себе и своей роли]
```

**Разница с @-командами:**
- `.cursorrules` — AI читает ВСЕГДА, автоматически, при каждом запросе
- `@File`, `@Codebase` и т.д. — AI читает только когда ты явно подключаешь

## Шаг 15. Базы знаний > промпты

Это главное правило модуля.

**❌ Длинный промпт:**
Каждый раз пишешь 10 строк контекста. Забываешь, путаешь, тратишь время. AI «забывает» в длинных диалогах.

**✅ База знаний:**
Один раз настроил .cursorrules. Один раз создал папку с контекстом. Пишешь короткий запрос — получаешь шедевр.

**Настрой контекст один раз — используй бесконечно.**

---

## Шаг 16. Частые ошибки

- **❌ Пишешь контекст в каждом промпте** — вместо этого создай файл и подключай через `@`
- **❌ Не создаёшь базу знаний** — AI каждый раз начинает с нуля
- **❌ Файлы называются «Без имени» и «file1»** — AI не понимает что где
- **❌ Нет README в папках** — AI не знает, что там лежит и зачем
- **❌ Грузишь всё сразу** — начни с личного, потом бизнес, потом проект
