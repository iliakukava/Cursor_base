# TeleFlow — честная оценка проекта и презентации

Это ревью написано перед защитой, чтобы ты вышел на питч уверенно: ясно видел свои сильные стороны, знал слабые места заранее, не поплыл на вопросах и понимал, что именно стоит сказать вслух, а что — держать в уме.

Оценки — мои, но с конкретными ссылками на код. Ревьюер — ИИ, который читал весь `src/teleflow/*.py`.

Версия ревью актуальна после добавления фич: `fingerprint`-дедуп, `/list`/`/rm`/`/mv`/`/undo`, стили `/write`.

---

## TL;DR

- Проект объективно **заметно выше среднего** для учебного финала: ~1.5 k строк Python в 16 модулях, реальная интеграция с двумя API (Telegram MTProto + OpenRouter), здравая архитектура с разделением ответственности, осознанные ограничения (батчи, тайм-бюджеты, whitelist, DRY_RUN, FloodWait-handling), **настоящий CRUD поверх базы знаний** с защитой от path-traversal и дедупликацией через SHA-256.
- Презентация теперь **editorial deep-blue**: асимметричные сетки, Instrument Serif + Inter + JetBrains Mono, римская нумерация, hairline-линии, film-grain. Выглядит как разворот журнала, не как AI-шаблон. Все плейсхолдеры заполнены, все 8 команд отражены, микро-CRUD описан отдельной секцией.
- Главный оставшийся риск на защите — **тесты и evals**. Теперь, когда есть чистые функции (`fingerprint`, `ranker`, `normalize_entry_id`), их покрытие — самая дешёвая следующая ценность.

## Как я смотрю на проект

### Что сильного (пиши это на слайде)

| Что | Где в коде | Почему это сильно |
|---|---|---|
| Реальный Telethon userbot | `src/teleflow/telegram_client.py` | MTProto, папки, `GetDialogFilters`, `GetPeerDialogs`, FloodWait/PeerFlood. |
| Разделение на слои | 16 модулей, один файл — одна ответственность | `collector`, `summarizer`, `ranker`, `publisher`, `knowledge`, `write_synthesizer`, `fingerprint`. |
| Боевые ограничения | `openrouter_timeout_sec`, `openrouter_annotate_budget_sec`, `run_once_budget_sec`, `dedupe_text_max_chars` | Видно, что думал про деньги, время, сеть и память. |
| Безопасность и доступ | `ALLOWED_USER_IDS`, `DRY_RUN`, `.env` с API-ключами | Whitelist на уровне обработчика, DRY_RUN по умолчанию. |
| Стабильность длинных ответов | `send_text` с chunked `_split_for_telegram`, `FloodWait` / `PeerFlood` | Реальная работа со стороной Telegram. |
| Атомарная запись данных | `_atomic_write_json` в `knowledge.py` | `tempfile.mkstemp` + `os.replace` — не MVP-уровень. |
| **Дедупликация на двух слоях** | `fingerprint.py` + `collector.dedupe_candidates_by_fingerprint` + `save_knowledge_entry` | SHA-256 от нормализованного текста, переиспользуется в дайджесте и в базе. |
| **Микро-CRUD поверх базы** | `handlers.py`: `/list`, `/rm`, `/mv`, `/undo` | `shlex.split` для парсинга, id-regex против path-traversal, per-user `/undo` через SQLite. |
| **Стили `/write`** | `config._parse_write_styles` + `write_synthesizer.synthesize_post(style_key=…)` | Конфигурируемая карта через `.env`, без правок кода — меняй стили и рестартуй. |
| Гибридный ранжировщик | `ranker.py` | `importance × 10 + metric_score` — работает даже без LLM. |
| Конфигурируемость | `config.py`: ~40 env-переменных, `_parse_*` валидация | Легко поменять модель, бюджет, папки, темы, стили без кода. |
| Поддержка MTProxy/SOCKS5 | `_build_client_kwargs` | Реалистично для Telegram в РФ. |
| Документация | `README.md`, `SPEC.md` v2.1 | Версионированная спека, smoke-чеклист, troubleshooting. |
| Дизайн презентации | `presentation/*` | Editorial deep-blue, не AI-шаблон: уникальные лейауты на каждом слайде. |

### Что слабого (я держу это при себе, но знаю)

| Что | Почему это риск на защите |
|---|---|
| **Нет автотестов** | На вопрос «как вы проверяете?» — пока только smoke-чеклист + DRY_RUN. Это теперь самый слабый пункт, потому что все «нет CRUD, нет дедупа» уже закрыты. |
| **Нет evals** | Качество LLM-разметки меряется глазами. Можно дать честный ответ — «планирую тест-сет, уже есть куда класть». |
| **`_iter_all_entries` O(N) на save/rm/mv** | При ~100 записях всё ок, при ~10 000 будет медленно. Решение — индекс `fingerprint → path` в `state.db`. Стоит упомянуть, если спросят про производительность. |
| **`/undo` без TTL** | Сохранил вчера, откатил сегодня — удалит. Исправляется добавлением timestamp в ключ или отдельного столбца. |
| **FSM выбора темы in-memory** | После рестарта демона «висящий forward» теряется. Но это эфемерное состояние, и это нормально. |
| **Нет CI/линтера в репо** | Не фатально, но спросят — скажи «инфраструктура — следующий шаг». |
| **NOISE_MARKERS захардкожены** | Можно вытащить в конфиг, один коммит. |

### Что стоит показать голосом

1. **Где реально ИИ** — только в двух точках: `DigestAnnotator` и `synthesize_post`. Остальное (ranker, dedupe, fsm) — детерминированные правила. Это честно и это сильно.
2. **Fingerprint-дедуп** — простая идея с большой пользой. SHA-256 нормализованного текста, один модуль, работает и в `/digest`, и при `save`. Это тебе дают «очки зрелости».
3. **Стили `/write`** — демонстрация, что LLM-часть конфигурируется без кода.
4. **Почему userbot** — Bot API не видит папки. Для B2B-версии userbot меняется на Bot API именно потому, что там нужна массовая раздача.
5. **`/undo` через SQLite** — не in-memory, переживает рестарт.

## Оценка презентации

### Что было и что стало

**Было** (до редизайна):
- Зелёная accent-палитра на тёмно-серой подложке, bento-плитки, KPI-кубики, speaker-timer.
- 12 слайдов на scroll-snap, прогресс-бар.
- Работало, но визуально выглядело «как сгенерировано ИИ»: узнаваемый AI-шаблон с типовыми лейаутами и яркими градиентами.

**Стало** (редизайн):
- **Deep-blue editorial-палитра.** Базовый тон — `#070b18`, тонкие оттенки до `#1a2448`. Один акцент — `#7befd9` (аквамарин), вспомогательные — `#e9d6a6` и `#5aa6ff`.
- **Serif display-шрифт** `Instrument Serif` для заголовков и крупных цитат. Sans — `Inter`, моно — `JetBrains Mono`.
- **Уникальный лейаут на каждом из 12 слайдов.** Не один паттерн «заголовок → сетка»: эссе с буквицей и pull-quote, таблица команд, лаб-записки, big-quote с marginalia, time-table, последняя полоса издания.
- **Римская нумерация** слайдов I–XII на левой вертикальной рельсе.
- **Тонкие hairline-линии** и минимум заливок — журнальная плотность.
- **Film-grain-оверлей** через SVG-шум — неброский, но сразу ломает ощущение «пластиковой» AI-графики.
- **Слайд «Product» обновлён** под новые 8 команд, сгруппированных: основные vs микро-CRUD.
- **Слайд «AI» обновлён** под четыре лаб-записки: DigestAnnotator, WriteSynthesizer, Ranker, Fingerprint-dedupe.

### Что меняю в прокачке

1. Полный редизайн в editorial deep-blue стиле.
2. 12 слайдов — каждый с уникальным лейаутом, не типовая сетка.
3. Слайд «Product» пересобран под 8 команд с группировкой.
4. Новый слайд (в разделе «AI») — про fingerprint-дедуп.
5. Роман-цифры на левой рельсе вместо счётчика стрелочек.
6. Счётчик слайдов сохранён в правом верхнем — для ориентации в темпе.
7. Таймер спикера удалён, личная информация удалена.
8. Film-grain и hairline-эстетика — анти-AI маркер.

## Список решений, которые я принял за тебя

- **Позиционирование** — «инструмент, который возвращает час в день, и каркас для B2B-продукта». Ложится под АССИСТ+.
- **Цифры экономии** — осторожные диапазоны, без «×10». В `speaker_notes.md` — с пометкой «по моим замерам».
- **Тон оффера** — прямой, короткий: «хочу в команду, приношу рабочий прототип».
- **Дизайн** — отказ от AI-шаблонов (bento, градиентные KPI-кубы, эмодзи), в пользу editorial-стиля журнала.
- **Бренд на слайдах** остаётся TeleFlow.
