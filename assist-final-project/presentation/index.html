    Tele Flow.

    Userbot, который возвращает час в день — собирает дайджест Telegram, ведёт тематическую базу и пишет черновики постов за автора.

    Стек
    Telethon · OpenRouter · SQLite · JSON
    Команды
    8 · включая микро-CRUD над базой
    Код
    15 модулей · ~1.5k строк Python
    Статус
    MVP в работе

    ↓ → Space — дальше · F — full screen

    № 02 · Context Что болит

    У автора нет второго мозга.

    Telegram — основной канал знания в 2026-м. Но ни один из десятков твоих каналов не встанет в очередь и не попросит: «прочитай меня первым».

    День начинается одинаково: четыре папки, сотни непрочитанных сообщений, три из них — важные, два — гениальные, остальные — реклама и анонсы. Отделить одно от другого глазами занимает час-полтора каждый день.

    К вечеру в голове — обрывки идей, которые хотелось разобрать в пост для своего канала. Кнопок «сохранить в тему» у Telegram нет. Заметки делать некогда. Половина идей к утру забыта.

    Инструменты, которые у нас есть, не созданы для работы со своим вниманием. Они — для чтения ленты другого.
    № 03 · Solution Интерфейс

    Один бот в ЛС. Восемь команд. База у тебя дома.

    TeleFlow — локальный Telegram userbot на Telethon. Обзор непрочитанного, архив по темам, черновик поста — на одном экране, без отдельных UI.

    Основные
    /digest
    Дайджест непрочитанного из выбранных папок. LLM размечает батчами, отдаётся в ЛС.
    forward + тема
    Пересылка поста → выбор темы → запись в data/knowledge/.
    /write <тема>
    Синтез авторского поста только из материалов этой темы. Флаг --style — выбор тональности.
    /stats
    Сколько записей в базе по каждой теме.
    Микро-CRUD
    /list <тема>
    Последние записи: id + хвост текста.
    /rm <id>
    Удалить запись по id с защитой от path-traversal.
    /mv <id> <тема>
    Перенести в другую тему — атомарно.
    /undo
    Откат последнего save. Хранится per-user в SQLite.

    № 05 · Economics Польза в цифрах

    ИИ здесь — не для «вау».
    Он возвращает время.

    Цифры — мои замеры на трёх папках, нижняя граница. Не выдуманная рыночная статистика.

    • ≈ 1 ч
      в день собственноручной работы — обзор каналов и поиск идей для постов. было / для одного автора
    • ~30 ч
      в месяц экономии при трёх папках и одной теме собственного производства. замер · декабрь
    • ≈ 60 ₽
      себестоимость одного дайджеста через OpenRouter (GPT-4o-mini). ≤ 80 постов в обработке
    • ×0
      копирайтеров на подбор источников — инструмент работает на контексте автора. стиль передаётся через флаг --style

    Не «сэкономил время». Вернул его — туда, где автор думает, а не копирует между вкладками.

    № 06 · Intelligence Где ИИ, а где правила

    Две точки вызова LLM. Всё остальное — детерминировано.

    § 01 · digest annotator

    Разметка дайджеста

    Батчи по 8 постов, обрезка входа, общий бюджет. Ответ — строго JSON; при сорванном формате есть парсер-fallback и дефолтный summary из текста.

    batch size
    8 posts
    text cap
    1 600 chars
    budget
    300 s
    temp
    0.2
    § 02 · write synthesizer

    Синтез авторского поста

    Читает только папку темы, режет контекст по лимитам и стилевому оверлею, просит «синтезируй одну идею, не пересказывай, не выдумывай вне входа».

    max files
    20
    context cap
    12 000 chars
    styles
    config · env
    temp
    0.35
    § 03 · hybrid ranker

    Рейтинг без магии

    final = importance × 10 + log₁(views, forwards, reactions) + keyword_hits. Если LLM сбойнёт на всём батче — дайджест всё равно выйдет.

    dependency
    LLM · optional
    fallback
    метрический скор
    § 04 · text fingerprint

    Дедупликация поверх всей базы

    SHA-256 по нормализованному тексту (без пробелов, эмодзи и модификаторов). При save сверяется по всем темам — дубликат не записывается.

    hash
    sha-256
    norm
    strip + noemoji
    reuse
    в digest дедупе тоже
    LLM — там, где нужен смысл правила — там, где нужна предсказуемость model: openai/gpt-4o-mini · env-configurable
    № 07 · Discipline Что не даёт упасть

    Взрослые привычки в учебном проекте.

    Всё продумано заранее, а не исправлено постфактум. Продакшен-ограничения — часть дизайна MVP.

    • Whitelist пользователейПервая проверка в обработчике сообщений — ALLOWED_USER_IDS.
    • DRY_RUN по умолчаниюНи канал, ни state-db не обновляются, пока автор явно не разрешит.
    • Chunked-отправкаДлинные дайджесты режутся по Telegram-лимиту с паузами между кусками.
    • FloodWait & PeerFloodОба исключения обрабатываются — процесс не падает на лимитах.
    • Atomic JSON writetempfile.mkstemp + os.replace — запись пережевёт краш в середине.
    • Тайм-бюджеты на всех слояхconnect, RPC, LLM-аннотация, весь /digest — у каждого свой лимит.
    • SOCKS5 и MTProxyРеальные сценарии для Telegram в РФ — два протокола прокси из коробки.
    • Защита от path traversalID — только [a-f0-9]{32}; /undo не выйдет за data/knowledge/.
    № 08 · Course map Курс → артефакты

    Ни один модуль
    не остался теорией.

    У каждого модуля курса — свой след в репозитории. Это осознанный выбор, а не случайность.

    M 00 · онбординг Setup Cursor, модели, токены, дисциплина коммитов.
    M 01 · старт First loop Скелет репозитория, первый python -m teleflow.
    M 02 · контекст The SPEC SPEC.md v2.1 + .env как явный контракт, .mdc-стиль правил.
    M 03 · инструменты Gitflow Маленькие коммиты, PR, ветки dev, 20+ осознанных шагов.
    M 04 · агенты Two agents DigestAnnotator и synthesize_post — отдельные роли.
    M 05 · telegram Userbot Telethon, FSM, 8 команд, chunked-send, прокси.
    M 06 · api OpenRouter chat/completions на stdlib urllib, без SDK.
    M 07 · вайб-кодинг This deck HTML / CSS / JS без сборки и npm. То, что вы видите сейчас.
    № 09 · Log Как прошли 40 часов

    ≈ 40 часов.
    Без AI — недели.

    И самое интересное: описание SPEC заняло больше, чем интеграция с OpenRouter. Контекст оказался дороже кода.

    phase 01 SPEC, контекст, границы MVP ~ 6 h
    phase 02 Telethon · папки · сбор непрочитанного ~ 12 h
    phase 03 OpenRouter, батчи, ranker, publisher ~ 9 h
    phase 04 FSM · knowledge · chunked-send · UX ~ 8 h
    phase 05 Упаковка · SPEC · deck · текст питча ~ 5 h
    × 5 ускорение / ручной путь 0 строк без ревью в Cursor 15 модулей · ~1.5 k lines
    № 10 · Horizons Что дальше

    От MVP
    к платформе.

    Архитектура разложена на слои так, что каждый горизонт можно запускать независимо.

      Планировщик дайджеста по расписанию, шаблоны рубрик для /write, переезд с ПК на VPS с systemd и Docker. Evals и юнит-тесты — отдельным модулем.

      Переход на Bot API, мультитенант, биллинг и per-tenant лимиты токенов, библиотека промптов под ниши.

      средний · продукт

    3. Платформа бизнес-ассистентов

      Маркет агентов от выпускников АССИСТ+, голосовой инбокс как отдельный канал, A/B моделей через OpenRouter.

      дальний · масштаб
