    что это

    Один userbot. Четыре команды. Полный контроль.

    Локальный Telegram userbot на Telethon: команды в ЛС, сохранение материалов по темам, генерация текста через OpenRouter.

    • /digest — дайджест непрочитанного по выбранным папкам
    • Forward поста → выбор темы → запись в локальную базу знаний
    • /write <тема> — синтез авторского поста из сохранённого
    • /stats — сколько постов уже лежит по каждой теме

    как это работает

    Схема потока данных

    где реально ИИ

    Две точки вызова LLM. Всё остальное — детерминированные правила.

    точка 1 · разметка дайджеста

    DigestAnnotator

    Батчами по 8 постов, с лимитом 1600 симв. на пост, общим бюджетом 300 с. Ответ строго JSON: keep, importance, category, summary. Если модель сорвала JSON — fallback на короткую выжимку из текста поста.

    точка 2 · синтез поста

    synthesize_post

    На /write <тема> читает только data/knowledge/<тема>/, режет до 20 файлов и 12 000 симв., идёт в LLM с инструкцией «синтезировать, не пересказывать, не выдумывать факты вне входа».

    • model · openai/gpt-4o-mini (можно сменить через env)
    • timeout · 120 s
    • batch · 8
    • annotate budget · 300 s
    • гибридный ранжировщик = LLM + log-метрики канала

    контроль и безопасность

    Взрослые привычки в учебном проекте.

    • Whitelist пользователей по ALLOWED_USER_IDS
    • DRY_RUN — превью без записи в канал и state
    • Chunked-отправка длинных сообщений (лимит Telegram)
    • FloodWait и PeerFlood обработаны, не падаем
    • Атомарная запись JSON через tempfile + os.replace
    • Тайм-бюджеты на connect / RPC / весь /digest
    • SOCKS5 и MTProxy — realistic под Telegram в РФ
    • Секреты только в .env, данные локально в data/
