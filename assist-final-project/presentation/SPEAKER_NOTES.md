# TeleFlow — спикерские заметки (6 минут)

## Слайд 1. Титул (0:00-0:30)

TeleFlow — это мой локальный AI-ассистент для Telegram.
Он решает практическую задачу: из перегруженного потока каналов делает структурированную базу идей и помогает быстро собрать черновик поста.

## Слайд 2. Проблема (0:30-1:00)

Основная боль: каналов много, непрочитанное копится, полезное теряется.
Чтобы сделать выжимку вручную, нужно тратить время каждый день.
Плюс контент для собственных публикаций приходится собирать вручную из разных мест.

## Слайд 3. Что это (1:00-1:35)

Это Telegram userbot на Telethon, который работает в личных сообщениях.
Ключевые команды:
- `/digest` — собрать дайджест;
- forward поста + выбор темы — сохранить в базу;
- `/write <тема>` — сгенерировать черновик поста;
- `/stats`, `/list`, `/rm`, `/mv`, `/undo` — управление базой.

## Слайд 4. Поток (1:35-2:15)

Логика простая:
папки Telegram -> сбор постов -> LLM-разметка через OpenRouter -> preview/канал -> база тем -> синтез текста.
ИИ встраивается только туда, где реально экономит время: фильтрация и генерация.

## Слайд 5. Польза (2:15-2:45)

Практический результат:
одним инструментом я получаю обзор непрочитанного, тематический архив и быстрый черновик для публикации.
Это снижает ручную рутину и ускоряет цикл «прочитал -> сохранил -> написал».

## Слайд 6. Контроль и безопасность (2:45-3:20)

В MVP важен контроль:
- whitelist пользователей;
- `DRY_RUN` для безопасного предпросмотра;
- ограничения на батчи/таймауты LLM;
- chunked-отправка длинных сообщений;
- локальное хранение данных (`JSON` + `state.db`).

## Слайд 7. Демо (3:20-4:10)

Демо-цепочка:
1) `python -m teleflow --daemon`
2) `/digest`
3) пересылаю пост и выбираю тему
4) `/write ai-агенты для бизнеса`
5) `/stats` или `/list`

Это показывает полный рабочий цикл продукта за 1-2 минуты.

## Слайд 8. Стек (4:10-4:40)

Почему такой стек:
- Python + Telethon для прямой работы с Telegram-папками;
- OpenRouter для гибкой замены моделей;
- JSON + SQLite как простой и прозрачный storage для MVP;
- FSM в ЛС вместо тяжёлого UI на первом этапе.

## Слайд 9. Время на проект (4:40-5:05)

Подставьте честную цифру/диапазон часов.
Полезно отдельно озвучить, что больше всего заняло время:
стабильность Telegram-сессии, предсказуемость LLM и UX команд.

## Слайд 10. Развитие (5:05-5:30)
